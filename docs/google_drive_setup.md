# Google Drive API 설정 및 `credentials.json` 발급 가이드

구글 드라이브 업로드 기능을 사용하기 위해 필요한 `credentials.json` 파일을 발급받는 절차입니다.

## 1단계: 구글 클라우드 프로젝트 생성
1. [Google Cloud Console](https://console.cloud.google.com/)에 접속합니다.
2. 상단 네비게이션 바에서 프로젝트 선택 드롭다운을 클릭하고 **'새 프로젝트'**를 선택합니다.
3. 프로젝트 이름(예: `BlogAutoPost`)을 입력하고 **'만들기'**를 클릭합니다.
4. 알림 창에서 프로젝트가 생성되면 **'프로젝트 선택'**을 클릭하여 해당 프로젝트로 이동합니다.

## 2단계: Google Drive API 활성화
1. 좌측 메뉴에서 **'API 및 서비스' > '라이브러리'**로 이동합니다.
2. 검색창에 `Google Drive API`를 입력하고 검색 결과에서 클릭합니다.
3. **'사용'** 버튼을 클릭하여 API를 활성화합니다.

## 3단계: OAuth 동의 화면 구성
1. 좌측 메뉴에서 **'API 및 서비스' > 'OAuth 동의 화면'**으로 이동합니다.
2. User Type을 **'외부(External)'**로 선택하고 **'만들기'**를 클릭합니다.
3. **앱 정보** 입력:
   - 앱 이름: `BlogAutoPost` (원하는 이름)
   - 사용자 지원 이메일: 본인 이메일 선택
4. **개발자 연락처 정보**: 본인 이메일 입력
5. **'저장 후 계속'**을 클릭합니다.
6. **범위(Scopes)** 단계는 건너뛰고 **'저장 후 계속'**을 클릭합니다.
7. **테스트 사용자 (Test Users)** 단계:
   - **'ADD USERS'** 버튼을 클릭하고 본인의 구글 계정(이메일)을 추가합니다. (이 단계가 중요합니다! 테스트 단계이므로 본인 계정을 등록해야만 업로드가 가능합니다.)
   - **'저장 후 계속'**을 클릭합니다.

## 4단계: 사용자 인증 정보(Credentials) 만들기
1. 좌측 메뉴에서 **'API 및 서비스' > '사용자 인증 정보'**로 이동합니다.
2. 상단의 **'+ 사용자 인증 정보 만들기'**를 클릭하고 **'OAuth 클라이언트 ID'**를 선택합니다.
3. **애플리케이션 유형**에서 **'데스크톱 앱'**을 선택합니다.
4. 이름은 기본값 그대로 두거나 원하는 대로 입력하고 **'만들기'**를 클릭합니다.
5. 'OAuth 클라이언트 생성됨' 팝업창이 뜨면, **'JSON 다운로드'** 버튼을 클릭합니다.

## 5단계: 파일 저장
1. 다운로드 받은 JSON 파일의 이름을 `credentials.json`으로 변경합니다.
2. 이 파일을 프로젝트 폴더 `c:\study\autoPostFlow\` 안에 넣어주세요.
